HTML tags

Though I have learned how to make web page in HTML, I didn't know what roles some tags actually play in the document. As I decided to study how to make web site, I took the first step with surfing some tags. So, LET'S BEGIN!

HURRAY!

DOCTYPE

Almost all of the HTML documents have "<!DOCTYPE html>" in its head. I also have seen it in VS Code snippet, but didn't know what role it plays.
In short, <!DOCTYPE html> tag informs browser of what version is the document writed in. <!DOCTYPE html>, in fact, is not HTML tag because it should be declared in front of <html>.

meta

If you use VS code and try to use snippet in it, you will see these three meta tags.

<meta charset="UTF-8" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<meta name="viewport" content="width=device-width, initial-scale=1.0"/>

<meta> tag is used to describe the contents of a Web page. These description is 'metadata', the data over the whole page. It should always be used in <head> tag.

<meta> tag mainly has 4 attributes, charset, http-equiv, content, name.
-- 'charset' attribute tells browser how to encode (or just show) the contents. UTF-8 is an abbreviation of 'Universal Coded Character Set + Transformation Format - 8bit' and means using unicode.
-- 'http-equiv' attribute provides some information to web browser before the contents are loaded. As you can guess by the name 'http-equiv(alent)', value has to be specific http header. One of its many values, "X-UA-Compatible" sets version to interpret the page. This and 'name' attribute is closely related to the other, 'content'
-- 'content' attribute provides detailed information about 'http-equiv' and 'name'. If there is attribute 'http-equiv' and its value is 'X-UA-Compatible', content has to be 'IE=edge' which means using the most recent version of Internet Explorer.
-- 'name' attribute refers to setting and fix it by value of 'content'. HTML5 allows us to control viewport, area users can see in web page, with <meta> element.
